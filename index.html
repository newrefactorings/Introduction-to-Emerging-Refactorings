Replacement Refactorings

Replacement refactorings involve replacing one part of the code with another to improve design, performance, or maintainability. This can mean swapping out an old implementation for a new one that is more efficient or easier to understand.

  • Replace ‘expected’ attribute with try-catch
  • Replace Anonymous Class with Lambda Expression
  • Replace array with collection
  • Replace Array with Singleton Collection
  • Replace ArrayList with Primitive Array
  • Replace Collections.sort() with List.sort()
  • Replace customized methods with API
  • Replace for loop with foreach
  • Replace for with while
  • Replace HashMap with LinkedHashMap
  • Replace if with switch
  • Replace inner Class with Lambda Expression
  • Replace lambdas with method references
  • Replace lambda with method reference
  • Replace Primitive with Object
  • Replace Traditional Loops with Enhanced For Loops
  • Replace Wrapper Object with Primitive
  • Replacing Iteration-based retrieval with Switch Expression

Modification Refactorings

Modification refactorings involve changing the structure of existing code without changing its external behavior. This helps make the code more readable, flexible, and easier to maintain.

  • Switch library
  • Switch to Alternative Implementation (method)
  • Turn local variable into field
  • Reverse If condition logic
  • Increase Field Visibility
  • Loosen Parameter’s Type Limitations
  • Avoid unnecessary actions by precondition checking
  • Combine Enum with Comparator
  • Complex Refactoring
  • Convert Static Nested Class to Inner Class
  • Decrease Field Visibility
  • Simplify Source code by Reusing Existing Data Type
  • Parameterize links (extract common links as parameters for normal texts)

Addition Refactorings

Addition refactorings involve adding new functionality to the existing codebase. This might include introducing new classes, methods, or components to enhance the system without altering its existing behavior. The goal is to extend the system's capabilities in a way that is maintainable and follows good design principles.

  • Add Brackets
  • Add Delegation for Super Class’s Constructor
  • Add final keyword

  • Remove Brackets

  • Remove Duplicate Condition Checks
  • Remove Duplication in Constructor by delegating to another constructor

  • Remove Repetitive Checking
  • Remove suppress warning annotation

Merge Refactorings

Merge refactorings involve combining multiple elements in the codebase that have similar responsibilities or behavior. This helps reduce redundancy and improves code maintainability.

  • Merge Class
  • Merge Conditional
  • Merge Method
  • Merge Parameter
