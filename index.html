Split Refactorings

Split refactorings involve breaking up larger elements into smaller, more manageable pieces. This helps improve code organization and adheres to the Single Responsibility Principle.

Replacement Refactorings

Replacement refactorings involve replacing one part of the code with another to improve design, performance, or maintainability. This can mean swapping out an old implementation for a new one that is more efficient or easier to understand.

Modification Refactorings

Modification refactorings involve changing the structure of existing code without changing its external behavior. This helps make the code more readable, flexible, and easier to maintain.

Addition Refactorings

Addition refactorings involve adding new functionality to the existing codebase. This might include introducing new classes, methods, or components to enhance the system without altering its existing behavior. The goal is to extend the system's capabilities in a way that is maintainable and follows good design principles.

Remove Refactorings

Remove refactorings involve eliminating unnecessary code to simplify the codebase. This can help reduce complexity, improve readability, and make the code easier to maintain.

Merge Refactorings

Merge refactorings involve combining multiple elements in the codebase that have similar responsibilities or behavior. This helps reduce redundancy and improves code maintainability. For example, Merge Conditions refactoring.
